The AgentOS is a high-performance runtime for serving agents, multi-agent teams, and agentic workflows in production. It provides:

1. SSE-compatible API endpoints that you can use to build your agentic product.
2. A powerful control plane for monitoring, managing and testing your agentic system.

Under the hood, the AgentOS is a FastAPI application that provides endpoints for serving, monitoring and managing your agentic system. Let's build our first AgentOS to see how it works, update the `agno_mcp.py` file to:

```python agno_mcp.py lines
from contextlib import asynccontextmanager

from agno.agent import Agent
from agno.db.sqlite import SqliteDb
from agno.models.anthropic import Claude
from agno.os import AgentOS
from agno.tools.mcp import MCPTools
from fastapi import FastAPI

# Create a database for the Agent
db = SqliteDb(db_file="agno.db")

# Define your Agent
agent = Agent(
    name="Agno Agent",
    model=Claude(id="claude-sonnet-4-0"),
    db=db,
    add_datetime_to_context=True,
    add_history_to_context=True,
    num_history_runs=3,
    markdown=True,
)

# Start the MCP connection in the FastAPI lifecycle
@asynccontextmanager
async def lifespan(app: FastAPI):
    global mcp_tools

    # Startup logic: connect to your MCP server
    agno_mcp_server = MCPTools(
        transport="streamable-http", url="https://docs-v2.agno.com/mcp"
    )
    await agno_mcp_server.connect()

    # Add the MCP tools to our Agent
    agent.tools = [agno_mcp_server]

    yield

    # Shutdown: Close MCP connection
    await agno_mcp_server.close()


# Create FastAPI app with lifespan
fastapi_app = FastAPI(lifespan=lifespan)

# Create your AgentOS
agent_os = AgentOS(agents=[agent], fastapi_app=fastapi_app)
app = agent_os.get_app()

if __name__ == "__main__":
    agent_os.serve(app="agno_mcp:app", reload=True)
```

<Tip>

Unfortuntately, the MCP server connection needs to be established in the FastAPI lifecycle.

</Tip>

    ```shell
    python agno_mcp.py
    ```

  </Step>

This will start your AgentOS on `http://localhost:7777`. Next, let's connect to the AgentOS control plane. Checkout the API endpoints at [localhost:7777/docs](http://localhost:7777/docs).

## Connect to the Control Plane

The AgentOS control plane is a web interface for monitoring, managing and testing your AgentOS runtimes. Open [os.agno.com](https://os.agno.com) and sign in to your account.

1. Click on the team/organization dropdown in the top navigation bar
2. Click the **"+"** (plus) button next to "Add new OS"
3. The "Connect your AgentOS" dialog will open
4. Select **"Local"** to connect to an AgentOS running on your local machine.
5. Enter the endpoint URL of your AgentOS. The default is `http://localhost:7777`.
6. Give your AgentOS a descriptive name like "Development OS" or "Local 7777".
7. Click the **"CONNECT"** button.
