Tools make agents - "agentic" by enabling them to interact with external systems like searching the web, running SQL, sending an email or calling APIs.

Agno comes with 80+ pre-built toolkits, but in most cases, you will write your own tools. The general syntax is:

```python
import random

from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.tools import tool


@tool(stop_after_tool_call=True)
def get_weather(city: str) -> str:
    """Get the weather for a city."""
    # In a real implementation, this would call a weather API
    weather_conditions = ["sunny", "cloudy", "rainy", "snowy", "windy"]
    random_weather = random.choice(weather_conditions)

    return f"The weather in {city} is {random_weather}."


agent = Agent(
    model=OpenAIChat(model="gpt-4o-mini"),
    tools=[get_weather],
    markdown=True,
)
agent.print_response("What is the weather in San Francisco?", stream=True)
```

<Tip>

In the example above, the `get_weather` function is a tool. When it is called, the tool result will be shown in the output.

Then, the Agent will stop after the tool call because we set `stop_after_tool_call=True`.

</Tip>

### Using the Toolkit Class

The `Toolkit` class provides a way to manage multiple tools with additional control over their execution. You can specify which tools should stop the agent after execution and which should have their results shown.

```python
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.tools.googlesearch import GoogleSearchTools

agent = Agent(
    model=OpenAIChat(id="gpt-4.5-preview"),
    tools=[
        GoogleSearchTools(
            stop_after_tool_call_tools=["google_search"],
            show_result_tools=["google_search"],
        )
    ],
)

agent.print_response("What's the latest about gpt 4.5?", markdown=True)
```

In this example, the `GoogleSearchTools` toolkit is configured to stop the agent after executing the `google_search` function and to show the result of this function.

## Function Execution Results

Tools return results through the `FunctionExecutionResult` class internally.

<Snippet file="function-execution-result-reference.mdx" />

### Simple String Returns

Most tools can simply return a string:

```python
@tool
def get_weather(city: str) -> str:
    """Get the weather for a city."""
    return f"The weather in {city} is sunny."
```

### `ToolResult` for Media Content

When your tool needs to return media artifacts (images, videos, audio), use `ToolResult`:

<Snippet file="tool-result-reference.mdx" />

```python
from agno.tools.function import ToolResult
from agno.media import ImageArtifact

@tool
def generate_image(prompt: str) -> ToolResult:
    """Generate an image from a prompt."""
    
    # Create your image (example)
    image_artifact = ImageArtifact(
        id="img_123",
        url="https://example.com/generated-image.jpg",
        original_prompt=prompt
    )
    
    return ToolResult(
        content=f"Generated image for: {prompt}",
        images=[image_artifact]
    )
```

<Note>
### How `ToolResult` Works Internally

When you return a `ToolResult`, Agno automatically:

1. **Extracts the content** as the main tool output
2. **Transfers media artifacts** to the `FunctionExecutionResult`
3. **Makes media available** to the LLM model

Here's what happens behind the scenes:

```python
# Your tool returns ToolResult
tool_result = ToolResult(
    content="Image generated successfully",
    images=[image_artifact]
)

# Agno internally processes this:
# 1. function_call_output = tool_result.content
# 2. function_execution_result.images = tool_result.images
# 3. Media gets attached to the conversation
```
</Note>

### Session State Management
The `FunctionExecutionResult` class plays a crucial role in maintaining session state across tool calls. When your tool modifies the `session_state` parameter, those changes need to be persisted back to the agent's session. Here's how it works:
```python
@tool
def update_user_profile(
    name: str,
    email: str,
    session_state: Dict[str, Any]
) -> str:
    """Update user profile information."""
    # Modify session state
    session_state["user_profile"] = {
        "name": name,
        "email": email,
        "updated_at": datetime.now().isoformat()
    }
    # Return simple string - Agno handles session state automatically
    return f"Updated profile for {name}"
```
Behind the scenes, Agno:
1. **Detects session state changes** in your tool function
2. **Captures the modified session_state**
3. **Creates a FunctionExecutionResult** with `updated_session_state`
4. **Persists the changes** back to the agent
