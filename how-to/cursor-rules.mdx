Using Cursor or another AI coding assistant with Agno? A `.cursorrules` file helps your AI understand Agno's patterns and best practices—making development faster and more accurate.

## Why This Matters

Without proper context, AI assistants might suggest outdated patterns or incorrect syntax. With `.cursorrules`, your AI will:

- Use correct parameter names and patterns
- Build workflows with the right approach (Agent vs Team vs Workflow)
- Follow Agno's performance optimizations (~3μs agent instantiation)
- Suggest tools and patterns that work with Agno

## Quick Start: Building Agents

### Basic Agent

```python
from agno.agent import Agent
from agno.models.openai import OpenAIChat

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    instructions="Clear, specific instructions",
    markdown=True,
)

agent.print_response("Hello!", stream=True)
```

### Agent with Tools

```python
from agno.tools.duckduckgo import DuckDuckGoTools

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    tools=[DuckDuckGoTools()],
    show_tool_calls=True,
)
```

### Agent with Memory

```python
from agno.memory.db import MemoryDb
from agno.db.sqlite import SqliteDb

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    memory=MemoryDb(db=SqliteDb(db_file="memory.db")),
    add_history_to_context=True,
)
```

### Agent with Knowledge (RAG)

```python
from agno.knowledge import Knowledge
from agno.vectordb.pgvector import PgVector

knowledge = Knowledge(
    vector_db=PgVector(
        table_name="docs",
        db_url="postgresql://user:pass@localhost/db"
    )
)

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    knowledge=knowledge,
    search_knowledge=True,
)
```

## Choosing the Right Pattern

### Agent vs Team vs Workflow

**Use Agent** when:
- Single focused task
- One set of tools sufficient
- No collaboration needed

**Use Team** when:
- Multiple agents with different expertise
- Want autonomous coordination
- Task benefits from multiple perspectives

**Use Workflow** when:
- Need specific execution order
- Have conditional logic
- Want full control over interactions

## Building Multi-Agent Systems

### Team Coordination

```python
from agno.team import Team

researcher = Agent(
    name="Researcher",
    model=OpenAIChat(id="gpt-4o"),
    tools=[DuckDuckGoTools()],
)

writer = Agent(
    name="Writer",
    model=OpenAIChat(id="gpt-4o"),
)

team = Team(
    members=[researcher, writer],  # Note: use 'members'
    model=OpenAIChat(id="gpt-4o"),
    instructions="Coordinate research and writing",
)

team.print_response("Research AI trends and write a report", stream=True)
```

### Step-Based Workflows

```python
from agno.workflow import Workflow, Step

workflow = Workflow(
    name="Research Workflow",
    steps=[
        Step(name="Research", agent=researcher),
        Step(name="Write", agent=writer),
    ]
)

result = workflow.run("AI developments in 2025")
```

### Conditional Workflows

```python
from agno.workflow import Condition

def needs_fact_checking(step_input):
    content = step_input.previous_step_content or ""
    return "study" in content.lower() or "research" in content.lower()

workflow = Workflow(
    steps=[
        Step(name="Research", agent=researcher),
        Condition(
            evaluator=needs_fact_checking,
            steps=[Step(name="Fact Check", agent=fact_checker)]
        ),
        Step(name="Write", agent=writer),
    ]
)
```

## Advanced Patterns

### Custom Tools

```python
from agno.tools import Toolkit

class WeatherTool(Toolkit):
    def __init__(self):
        super().__init__(name="weather", tools=[self.get_weather])
    
    def get_weather(self, city: str) -> dict:
        """Get current weather for a city.
        
        Args:
            city: City name
        """
        # Implementation
        return {"city": city, "temp": 72}

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    tools=[WeatherTool()],
)
```

### Structured Output

```python
from pydantic import BaseModel, Field

class ResearchOutput(BaseModel):
    summary: str
    key_points: list[str]
    confidence: float = Field(ge=0, le=1)

result = agent.run(
    "Research AI trends",
    response_model=ResearchOutput
)
```

## Common Mistakes

### Creating Agents in Loops

<CodeGroup>
```python ❌ Don't do this
for query in queries:
    agent = Agent(model=OpenAIChat(id="gpt-4o"))
    agent.run(query)
```

```python ✅ Do this instead
agent = Agent(model=OpenAIChat(id="gpt-4o"))
for query in queries:
    agent.run(query)
```
</CodeGroup>

### Too Many Tools

<CodeGroup>
```python ❌ Don't do this
agent = Agent(
    tools=[tool1, tool2, ..., tool20]  # Too many!
)
```

```python ✅ Do this instead
agent = Agent(
    tools=[search, calculator, emailer]  # Focused set
)
```
</CodeGroup>

### Vague Instructions

<CodeGroup>
```python ❌ Don't do this
agent = Agent(
    instructions="Help the user"
)
```

```python ✅ Do this instead
agent = Agent(
    instructions="""
    You are a technical documentation expert.
    - Analyze code and generate clear documentation
    - Follow Google-style docstrings
    - Include usage examples
    """
)
```
</CodeGroup>

### No Error Handling

<CodeGroup>
```python ❌ Don't do this
response = agent.run(query)
print(response.content)
```

```python ✅ Do this instead
from agno.exceptions import AgentException

try:
    response = agent.run(query)
    print(response.content)
except AgentException as e:
    logger.error(f"Error: {e}")
```
</CodeGroup>

## Testing

```python
import pytest

@pytest.fixture
def test_agent():
    return Agent(model=OpenAIChat(id="gpt-4o"))

def test_basic(test_agent):
    response = test_agent.run("What is 2+2?")
    assert "4" in response.content

@pytest.mark.asyncio
async def test_async(test_agent):
    response = await test_agent.arun("Hello")
    assert response is not None
```

## Production Deployment

```python
from agno.os import AgentOS
from agno.db.postgres import PostgresDb

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    db=PostgresDb(connection_url=os.getenv("DATABASE_URL")),
)

agent_os = AgentOS(agent=agent)
app = agent_os.get_app()
```

### Production Checklist

- ✅ Use PostgreSQL (not SQLite)
- ✅ Environment variables for secrets
- ✅ Error handling on all calls
- ✅ Logging configured
- ✅ Tests passing

## For Your Projects

Create a `.cursorrules` file in your project root:

```text .cursorrules
# My Agno Application

You are an expert in building agents with Agno.

## Project Overview
- Application: [Your app name]
- Purpose: [What it does]

## Agno Patterns
- Use Agent for focused tasks
- Use Team for autonomous coordination
- Use Workflow for sequential processes
- Always reuse agents (never create in loops)
- Use type hints and Pydantic models

## Development Standards
- Python 3.7+ with type hints
- Environment variables for secrets
- PostgreSQL for production
```
